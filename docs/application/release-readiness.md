# Release Readiness

Use this checklist before external review, demos, or maintainership-facing submissions.

## Documentation

- README reflects the current repo truthfully
- contributing and security docs are internally consistent
- demo commands still work
- roadmap language matches what is actually implemented

## Demo surfaces

- landing page builds and explains the current state clearly
- dashboard reflects the shared finding model
- exploit-lab still demonstrates attack flow coherently
- analyzer exports HTML and JSON without manual cleanup

## Repo maturity

- CI workflow files are present and match the local commands
- example fixtures are organized and reviewable
- later-phase placeholders are documented instead of implied to be complete
- plugin and advisory surfaces have an obvious local home
