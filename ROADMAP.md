# Sentinel Forge Roadmap

This roadmap tracks how Sentinel Forge moves from research posture to usable Soroban security infrastructure.

## Phase 0

Security research, Soroban ecosystem study, attack taxonomy, and MVP selection.

## Phase 1

Branding, monorepo setup, landing page, Rust workspace, and initial engine scaffolding.

## Phase 2

Documentation and architecture maturity:

- public README and roadmap
- contributor and security policy
- architecture diagrams and system docs
- research summaries and developer guides

## Phase 3

Static analyzer MVP:

- ingest and parsing pipeline
- intermediate representation
- first detector set
- evidence-rich reporting

## Phase 4

Visualization and tooling:

- dashboard for findings, severity views, scan history, and code-linked traces
- exploit-lab prototype for attack-path graphs, replay, and state transitions
- JSON, SARIF, and HTML reporting from the analyzer
- CI-friendly workflows and editor integration scaffolding

## Phase 5

Contributor readiness:

- repository-side issue and pull request templates
- stronger vulnerable, secure, benchmark, and exploit fixtures
- detector, analyzer, reporter, and engine extension guides
- contributor workflow, review standards, and testing expectations
- issue taxonomy, label guidance, and project-board planning

## Phase 6

Application preparation:

- public demos
- polished documentation
- maintainership-facing maturity improvements

## Phase 7

Post-application expansion into deeper execution systems, richer reporting, and integration surfaces.

## Phase 8

Security ecosystem scaling across research, automation, collaboration, and sustained Soroban coverage.
